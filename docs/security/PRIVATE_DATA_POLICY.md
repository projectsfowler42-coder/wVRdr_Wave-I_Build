# Private Data Policy

## Public-Safe Repo Material

Allowed in GitHub:

- source code
- sanitized schemas
- mock data
- tests
- UI components
- public-safe docs
- placeholder configs

## Private Vault Material

Keep outside Git:

- real API keys
- broker credentials
- financial account exports
- raw Mouseion/MSP datasets
- exact capital deployment plans
- proprietary strategy notes
- private prompt chains
- raw PDFs/DOCX/XLSX/ZIPs

## Operating Rule

No file enters Git unless it would be safe to show a future contractor, auditor, or investor without exposing personal finances, broker access, proprietary strategy edge, or private Mouseion/MSP material.
