# 2. Product Definition

## 2.1 Binding definition

**Wave-I is my capital inside the market.**

Wave-I is a manual compounding cockpit.

Wave-I must show:
- what I own
- what the market is doing to it
- what action that justifies now
- why I acted
- what happened after

Wave-I is not:
- a generic market dashboard
- a regime-theater dashboard
- a prop-firm simulator
- a broker execution console
- a social/news feed
- a hidden AI scoring surface
- the full WaveRider judgment engine

## 2.2 Binding page question sequence

Every primary Wave-I surface must answer, in order:

1. What is my capital doing?
2. What is the market doing to it?
3. What action does that justify now?
4. Why did I act, and what happened after?

If a proposed UI or feature cannot be mapped to one of those questions, it does not belong in Wave-I by default.

# 11. Data Architecture Authority

## 11.1 No-backend-now boundary

Wave-I must be fully usable now with no backend requirement.

Allowed now:
- manual mode
- mock mode
- CSV mode
- API placeholder mode
- local persistence
- local snapshots
- local decision logs
- local notes
- local quote snapshots
- deterministic calculators

Not required now:
- live broker auth
- scheduled jobs
- backend quote aggregation
- account sync
- push notifications
- cross-device sync

## 11.2 Binding provider rule

All data must flow through a provider boundary.

Required provider modes:
- `manual`
- `mock`
- `csv`
- `api` placeholder

No component should hardcode its own data path as final truth.

## 11.3 Binding storage rule

Wave-I must persist local truth durably.

Required persisted domains:
- holdings
- treasury balances
- transfers
- snapshots
- decision log
- notes
- settings
- optional local market context snapshots

Local persistence must survive reload and reopen.

# 12. Quote and Telemetry Authority

## 12.1 Quote role

Quotes are supporting context, not the structural center of the product.

The app must remain useful even when remote quote refresh fails.

## 12.2 Binding runtime lesson

A green deploy does not equal a working product.

A green Pages deployment with a broken runtime quote path is still a broken product.

## 12.3 Release-grade rule

Public browser CORS-proxy chains are not release-grade authority for the product architecture.

They may be used experimentally.
They may not define the release story.

## 12.4 Current acceptable posture

For this phase, quote context may come from:
- manual entry
- local snapshots
- graceful cached values
- optional refresh paths that can fail safely

If live quotes fail, the product must degrade gracefully rather than collapse into uselessness.
